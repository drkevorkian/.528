# AGENTS.md

## Cursor Cloud specific instructions

### Overview

This is a Rust workspace (28 crates) implementing the SRS Media System: a proprietary media codec/container ecosystem with CLI tools, a licensing server, desktop player, and admin UI. See `README.md` for full details.

### Build & Test

- **Build:** `cargo build --workspace`
- **Lint:** `cargo clippy --workspace` (passes clean; `cargo fmt --check` has pre-existing diffs in `tools/quality_metrics`)
- **Test:** `cargo test --workspace` (all tests pass; ~330 tests across all crates)
- **Toolchain:** `rust-toolchain.toml` pins `stable` with `rustfmt` + `clippy` components. Rustup auto-installs on first use.

### Running Services

**Licensing Server** (required for encode/decode/mux/import CLI operations):
```bash
mkdir -p var
SRS_CONFIG_PATH=config/srs.toml cargo run -p srs_license_server
```
Binds to `127.0.0.1:3000`. Health check: `curl http://localhost:3000/healthz`

**CLI (headless, no GUI needed):**
```bash
cargo run -p srs_cli -- <command>
```

### Licensing Gotchas

- CLI `encode`, `decode`, `mux`, `demux`, `import`, `transcode` commands require an **editor-entitled license key** (`--key <KEY>`).
- `analyze`, `codecs`, `play` commands work **without** a license key (play-only mode).
- To issue and activate a key for local development:
  1. Issue: `curl -s -X POST http://localhost:3000/issue -d 'email=dev@localhost'` — returns a key like `SRS-XXXX-XXXX-XXXX-XXXX`.
  2. Upgrade features: `curl -s -X POST http://localhost:3000/api/v1/admin/licenses/features -H 'Content-Type: application/json' -d '{"license_id":"<license_id>","features":["basic","editor_workspace","encode","decode","mux","demux","import","transcode"]}'`
  3. Use: `cargo run -p srs_cli -- --key "SRS-XXXX-XXXX-XXXX-XXXX" encode input output`
- The feature that grants editor mode is `editor_workspace` (not `editor`).

### GUI Apps

`srs_player` and `srs_admin` require a display server (X11/Wayland). They cannot run in headless Cloud Agent environments. Use CLI commands for testing.

### FFmpeg

Optional, feature-gated behind `--features ffmpeg` in `libsrs_compat`. Not needed for default builds or tests.

### Sample Files

Bootstrap samples can be regenerated per `samples/README.md`. The raw source files (`sample_video.raw`, `sample_audio.pcm16le`) are committed; derived files (`.srsv`, `.srsa`, `.528`) are generated by the CLI encode/mux pipeline.
